# e2e_yolo

ROS2 vision node running **yolo** (yolo_v8s) on **cuda**.

Generated by [vision-factory](https://github.com/robocore/vision-factory).

## Quick Start

```bash
# Install Python dependencies
pip install -r requirements.txt

# Build with colcon
cd /path/to/ros_ws/src
cp -r e2e_yolo .
cd ..
colcon build --packages-select e2e_yolo
source install/setup.bash

# Run
ros2 launch e2e_yolo vision.launch.py
```

## Docker

```bash
docker build -t e2e_yolo .
docker run --gpus all --rm e2e_yolo
```

## Configuration

Edit `config/params.yaml`:

```yaml
e2e_yolo_node:
  ros__parameters:
    input_topic: "/camera/image_raw"
    confidence_threshold: 0.25  # Minimum confidence for detections
    iou_threshold: 0.45  # IoU threshold for NMS
    classes: []  # Filter to specific class IDs (empty = all)
```

## Topics

### Subscribed
- `input_topic` (sensor_msgs/Image) — Input camera image

### Published
- `detections` (DetectionArray)
- `visualization` (sensor_msgs/Image) — Image with drawn bounding boxes

## Fine-tuned Models

To use your own fine-tuned weights, update `config/params.yaml`:

```yaml
e2e_yolo_node:
  ros__parameters:
    # Point to your custom weights:
    # weights_path: "/path/to/my_finetuned_model.pt"
```

The architecture code is identical — only the weights change.
